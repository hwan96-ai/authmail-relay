## Section 1 — Architecture

```
LIBRARY MODE                      HTTP MODE
caller → SmtpSender               caller → POST /send → FastAPI
       → Notifier                        → Pydantic validation
       → smtplib                         → HTTPBearer (hmac.compare_digest)
                                         → SmtpSender / Notifier
                                         → smtplib

CLIENT SDK MODE
caller → EmailServiceClient → httpx.Client → HTTP MODE
```

### Coupling concerns

- **Notifier base couples directly to `SmtpSender`** (`notifiers.py:48`). If a future
  provider abstraction (SendGrid, SES) is introduced, `Notifier.__init__` and
  `TemplateNotifier.__init__` would both need to accept a wider `EmailTransport`
  protocol. Not currently a problem — SMTP is the only transport — but worth flagging.
- **`create_app` mixes wiring + env-reading** (`api.py:107-128`). Library callers can
  inject everything (good), but the default path silently reads 5+ env vars. Acceptable
  for a thin service shim.
- **`TemplateNotifier` does not inherit from `Notifier`** (deliberate — its `send`
  signature uses `**context`, incompatible with the abstract `(to, name, payload)`
  contract). Documented implicitly via the file but not in the class docstring.

### Realistic production failure per integration point

| Integration              | Failure mode                                  |
|--------------------------|-----------------------------------------------|
| smtplib → SMTP server    | TLS handshake hang past `timeout=10` → 502    |
| FastAPI → SmtpSender     | sender returns False (refused dict) → 502     |
| httpx Client → FastAPI   | DNS failure on `email-service:8000` → raises  |
| Pydantic validation      | huge `html_body` (>10MB) — no size cap        |
| Env loading              | `SMTP_PORT="abc"` → ValueError at boot        |

## Section 2 — Code Quality

- **DRY**: `_no_crlf` exists in `api.py:25` AND `\r/\n` check inline in
  `sender.py:43-46`. **Acceptable defense-in-depth**: Pydantic gives 422 (caller
  error), sender gives False (logged). Both layers needed because the sender is
  also a public library API.
- **Naming**: ✅ Clear (`SmtpSender`, `MagicLinkNotifier`, `SendEmailRequest`).
- **Error handling**: ✅ `sender.py:84` catches broad `Exception` but logs via
  `logger.exception` — acceptable because the caller contract is `bool`. API layer
  re-raises as 502. No silent swallowing.
- **Over-engineering**: ❌ none. ~200 lines total of core logic.
- **Under-engineering**:
  - Missing **async client variant** (`AsyncEmailServiceClient`). Backend callers
    in async frameworks must run `EmailServiceClient.send` in a threadpool.
  - No **request size limit** on `html_body`.
  - No **structured retry** on transient SMTP errors (421, 451) — single attempt.

## Section 4 — Performance

- **Synchronous send blocks worker**: FastAPI runs `send_email` in a threadpool
  (sync def + smtplib), so each request occupies a worker thread for the full
  `timeout=10s` worst case. N concurrent senders contend for
  `--workers × thread_pool_size` slots.
- **Recommendations** (deferred — not in this branch):
  1. Run `uvicorn --workers 4` + tune `--limit-concurrency`.
  2. Add `POST /send/batch` to amortize TLS handshake across N recipients.
  3. Future: async client + `aiosmtplib` for true concurrency.

## Test Plan

### Affected Pages/Routes
- `POST /send` — main send endpoint
- `POST /send/magic-link` — templated magic link
- `POST /send/otp` — templated OTP
- `GET /health` — no-auth health check
- `GET /docs` — auto OpenAPI

### Key Interactions to Verify
- Bearer auth on all `/send/*` endpoints
- `X-Dry-Run` header short-circuits sender invocation
- CRLF in `to`/`subject`/`cc`/`bcc` → 422
- Empty `to`/`subject`/`html_body` → 422
- Magic-link without `MAGIC_LINK_BASE_URL` configured → 503

### Edge Cases
- `to` valid but SMTP refuses → 502
- Partial `cc` refusal → entire send returns False → 502
- SMTP timeout → caught → 502
- 100-recipient bcc list (no test today; recommended)
- `SMTP_PORT="abc"` at boot (no test today; recommended)
- `html_body > 10MB` (no test today; recommended)

### Critical Paths
- Caller backend → HTTP `/send` → SMTP → response
- Magic-link template URL encoding (`&`, `=`, `/`, `+` in token)
- OTP code HTML-escape (does not enforce alphanumeric — by design)
